Using Git in Command Line

Anne LoVerso (Olin College of Engineering)

What is Git

  • A version control system, free and open source
  • Initially designed and developed by Linus Torvalds for Linux

Get Git

sudo apt-get install git

git config

Located in ~/.gitconfig

git config --global user.name "Anne LoVerso"

git config --global user.email "aelv13@gmail.com"

git config --global color.ui true/false

Uses Vi as text editor, unless you have an override in .bashrc

git config --global core.editor nano

Vocabulary

  • commit - A revision, set of changes to file(s)
  • HEAD - The current working commit
  • remote - The version of something hosted externally (like on GitHub)
  • origin - default name for the remote you cloned from
  • master - The main branch you make changes in
  • upsteam - when branching or forking, the code's original version

Basic Git commands

git status
git log
git diff
git show
git clone [url]
git add [file]
git stash
git checkout [file]
git rm [file]
git commit

Opens a text editor for writing commit message

Elements of a good message:

  • Title line no more than 80 characters
  • Specifies what got changed
  • Imperative tense

Use -m option sparingly

git push
git pull
git add -i
